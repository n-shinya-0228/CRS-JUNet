# 研究の進め方
実行コマンド
--------------------------------------------------------------
- 学習

  `uv run python train.py --dataset SemanticKitti/ --arch_cfg config/arch/SJunNet.yaml --log logs/BEV_`

- グラフの確認

  `uv run python -m tensorboard.main --logdir logs --port 6006`

- 推論

  `uv run python infer_bev_512v3.py --dataset SemanticKitti/ --arch_cfg config/arch/SJunNet.yaml --data_cfg config/labels/semantic-kitti.yaml --pretrained logs/BEV_512_/best_val.path --save_path output/BEV_`

- 評価　

  `uv run python evaluate_iou.py --dataset SemanticKitti/ --predictions output/ --split valid --data_cfg config/labels/semantic-kitti.yaml`

- 結果の可視化

  `WAYLAND_DISPLAY="" uv run python vis_open3d.py --bin_path SemanticKitti/sequences/08/velodyne/000000.bin --label_path output/BEV_/sequences/08/predictions/000000.label`

- データの作成　

  `uv run python prepare_bev2.py --dataset SemanticKitti/ --data_cfg config/labels/semantic-kitti.yaml`

- データの削除
  
  `rm -rf SemanticKitti/sequences/*/bev`

バージョンの切り替え
--------------------------------------------------------------
- dataset 

  Parser.pyの中のfrom .SemanticKittiをかえる

- model 
  
  yamlファイルのmodel"name"のところをかえる（新しく作ったら__init__.pyに追加する）

- laserscan 
  
  prepare_bev.pyのところを変更して新しいデータを作成する（必要によってdatasetの中も変更）

- trainer 
  
  train.pyのところをかえる

**それぞれのバージョンによってyamlファイルの画像サイズやバッチサイズなどの数値をかえる**

それぞれのコードの役割
--------------------------------------------------------------
★dataset
- SemanticKitti.py
  
  このファイルは、生のデータをネットワークが理解できるテンソルに加工を担当している。

  PyTorchの Dataset クラスを継承しており、主に「1フレーム（1スキャン）分のデータをどう処理するか」というルールが書かれている。
  
  - データの読み込み: ハードディスクから生の3D点群データ（.bin）と正解ラベル（.label）を探して読み込みこむ。事前計算された .pt ファイルを読み込むこともある。
  - 2Dへの投影: 3Dの点群を、ネットワークが処理しやすい2Dの画像形式（球面投影のRange Image、あるいは上から見下ろしたBEV画像）に変換する。
  - 前処理とノイズ除去: レーザーが当たらなかった「黒抜け（小穴）」を周囲の値で埋めたり（インペインティング）、データのスケールを0〜1に揃えたり（正規化）する。
  - データ拡張: 学習時のみ、画像を反転させたり一部を隠したりして、モデルが景色を丸暗記（過学習）するのを防ぐ。

- Parser.py
  
  このファイルは、SemanticKitti.py が1つずつ作ったデータを、ネットワークの元へ効率よく運ぶのを担当している。

  PyTorchの DataLoader という機能を使って、学習ループへのデータの渡し方を管理している。

  - データセットの分割: 膨大なデータを「学習用（Train）」「検証用（Valid）」「テスト用（Test）」の3つのグループに正しく振り分ける。
  - バッチ化: ネットワークは1枚ずつではなく、複数枚（例：4フレームや8フレーム）を一度にまとめて処理します。Parserは、SemanticKittiが作った1フレーム分のデータを指定されたバッチサイズ分だけ集め、1つの巨大なテンソル（例：[4, 4, 512, 512]）に結合する。
  - シャッフル: 学習用データを取り出す順番を毎回ランダムにかき混ぜて、学習の偏りを防ぐ。
  - 特殊な結合ルール（Collate関数）: 画像テンソルは単純に結合し、文字列（ファイルパス）などはリストにまとめる、といった複雑な仕分け作業を collate_fn を使って安全に行う。

★model
- JunNet.py
  
  このファイルは、運ばれてきたデータから特徴を読み取り、「どこに何があるか」を判断するネットワーク（脳）の構造を担当している。

  - 情報の仕分け（Mask-Aware Stem）: 運ばれてきた6チャネルのデータのうち、5つの特徴量（高さや厚み）と1つのマスク（点があるかないか）を分け、マスクを使って「点がない場所」のノイズを強制的にシャットアウトする。
  - 特徴の濃縮（Encoder）: 入力された 512x512（または256x256）のデータを、ResStage というブロックを通して、解像度を下げながら情報の密度（チャネル数）を高めていく。
  - 核心的な理解（Bottleneck / Swin Attention）: 最も濃縮されたデータに対し、ASPP（広範囲を見る）や Swin Transformer（窓に区切って細部を見る）を使い、物体の境界や形を深く理解する。
  - 復元と判定（Decoder / Heads）: 濃縮された情報を元の解像度まで引き戻しながら、最終的に各ピクセルが「車」なのか「道路」なのかを 20クラスの確率（Logits）として出力する。
  - 境界の意識（Boundary Head）: 研究のポイントである「エッジ」を特別に意識させるための専用の出口（Head）を持ち、物体の輪郭をより鮮明に描き出す。

★utils
- laserscan.py
  
  このファイルは、3D点群をデジタルデータとして扱うための変換を担当している。

  - 3Dから2Dへの計算（投影）: x, y, zというバラバラな点の集まりを、パノラマ写真（Range Image）や地図（BEV）の「どのピクセルに置くべきか」を数学的に計算する。
  - 属性の整理: 1つの点に対して「距離」「反射強度」「座標」といった情報をセットにして保持し、画像化したときに「1ピクセルに複数の情報を重ねる（多チャネル化）」土台を作る。
  - ラベルの転写: 3Dの点についている正解ラベル（車、道路など）を、2D画像上の対応する位置に一寸の狂いもなくコピーする。
  - 極座標変換（Polar Projection）: 最新の SJunNet9 等に対応するため、これまでの四角い地図だけでなく、自分を中心に放射状に広がる「極座標」への変換ルールも保持している。
  
★trainer
- trainer.py

  このファイルは、モデルにデータを繰り返し見せ、間違いを指摘し、一歩ずつ正解へ導く「学習の総監督」を担当している。

  - 間違いの採点（Loss Function）: モデルが出した答え（予測）と正解を比べ、どれくらい間違っているかを計算する。

    - Focal Loss: 判定が難しい「珍しい物体」への間違いを厳しく採点する。
    - Lovasz Loss: 点群セグメンテーションで最も重要な指標である「mIoU（重なりの精度）」が直接良くなるように調整する。
    - Boundary Loss: 物体の「エッジ（境界線）」を間違えると大きなペナルティを与えることで、研究の肝である輪郭の鮮明さを叩き込む。
  - 復習のペース調整（Scheduler）: 最初はゆっくり（Warmup）、中盤は全力で、終盤は細かく丁寧に（Cosine Annealing）学習の歩幅（学習率）を変え、最も賢い状態（Best mIoU）を目指す。
  - 過去の自分を忘れない（EMA: 指数移動平均）:毎回の学習でフラフラせず、過去の賢かった自分の重みを少しずつ混ぜ合わせることで、安定した最強のモデル（Best Val）を作り上げる。
  - 成長の記録（TensorBoard）:学習中の Loss や精度の推移をグラフ化し、ブラウザからリアルタイムで「脳の成長具合」を観察できるようにデータを書き出す。
  - 混合精度学習（GradScaler）:性能を引き出すため、計算を少し簡略化（16bit）してスピードを上げつつ、重要な部分は精度を保つことで、学習時間を大幅に短縮する。

★その他
- prepare_bev.py
  
  このファイルは、BEVで学習時間を短縮するために事前に変換させるのを担当している。

  - 事前計算（Pre-computation）: 学習のたびに「3D → BEV」の重い計算を繰り返すと時間がもったいないため、事前に計算を終わらせている。
  - テンソル保存: 変換した BEV データを、PyTorch が最も高速に読み込める .pt 形式（テンソルファイル）でハードディスクに書き出す。
  - メモリ節約: gc.collect() などを使って、膨大なデータを処理する際にパソコンのメモリがいっぱいにならないよう掃除しながら作業する。

- infer.py
  
  このファイルは、学習を終えて賢くなったモデル（脳）を使って、実際の点群データに対する「本番のテスト解答（予測ラベル）」を作成し、保存する役割を担当している。

  - 記憶（チェックポイント）の呼び出し: --checkpoint 引数を通じて、Trainer が保存した best_val.path（Loss が一番低く、一番賢かった時の重みデータ）を読み込んでモデルにセットする。
  - 解答用紙（フォルダ）の準備: SemanticKITTI のデータ構造（シーケンス00〜21など）に合わせて、結果を書き込むための sequences/XX/predictions/ というフォルダ群を自動で一気に作成する。※古い予測結果が残っていたら、一度ゴミ箱に入れて綺麗にしてから作り直す。
  - 現場監督（Userクラス）への指示: 実際の「1フレームずつデータを読み込んで AI に見せ、答えを書き出す」という面倒な実作業は lib.user.py の User クラスに丸投げ（user.infer()）して実行させる。
  
- vis_open3d.py
  
  このファイルは、数字の羅列でしかないテスト結果（点群とラベル）を、人間が直感的にグリグリ動かして見られる「3D空間のビューアー（虫眼鏡）」を作る役割を担当している。

  - 色の辞書の読み込み: semantic-kitti.yaml から「車は青色」「道路は紫色」といったカラーマップ（ルール）を読み込む。
  - 点群とラベルの合体:
    - .bin ファイルから空間の「形（x, y, z）」を取り出す。
    - .label ファイルから AI が予測した「意味（ID）」を取り出す（下位16ビットだけを取り出して、余計なインスタンスIDを除去する安全設計）。
  - 色塗り作業: 点群の1粒1粒に対して、ラベルIDに基づいた「色（RGB）」を塗っていく。Open3Dの仕様に合わせて、色を0〜1の範囲に正規化している。
  - 3Dレンダリング（Open3D）: 塗った点群データを3D空間に配置し、マウスで回転・拡大縮小できるインタラクティブなウィンドウを立ち上げる。

Zone.Identifierの消し方
--------------------------------------------------------------
カレントディレクトリ以下にある `:Zone.Identifier` で終わるファイルを検索して表示 

`find . -type f -name '*:Zone.Identifier' -print`

確認後、問題なければ実際に削除 (-delete オプション)

`find . -type f -name '*:Zone.Identifier' -delete`

もしくは -exec rm {} \; を使う場合 (少し遅いが確実)

`find . -type f -name '*:Zone.Identifier' -exec rm {} \;`

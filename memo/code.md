**★Range Image（球面投影）**

バージョン,特徴・テーマ,距離の補間手法,ラベルの補間,ch数,長所,短所

- V1,力技の補間,全画面ガウシアンフィルタ,なし,5 ch,長: 隙間が埋まる,短: 処理が重く、空や背景まで無理やり埋めてノイズになる

- V2,補間廃止・マスク化,なし（0埋め）,なし,6 ch,長: 処理が最速。マスクで無効領域を教えられる,短: ネットワーク側の負担が大きい

- ★V3,小穴特化のスマート補間,縦横の小穴のみ1D線形補間,なし,6 ch,長: 物体表面の隙間だけ綺麗に埋まる,短: 特になし（優秀なベースライン）←2025先行研究で使用

- V4,反復膨張と擬似ラベル,近傍の平均と最大値のブレンド,3x3近傍の多数決,6 ch,長: 欠損ラベルも学習に使える,短: 背景と前景の境界が混ざってエッジが崩れる

- V5,エッジ保存（高精度版）,Joint Bilateralフィルタ,深度が最も近い画素からコピー,6 ch,長: 境界が崩れず、物理的にも非常に正確,短: 計算コストが高い

- V6,小物体エコヒイキ,クラスごとに独立して膨張,多数決（小物体クラスの票を倍増）,6 ch,長: バイクなどの少数クラスの精度が上がりやすい,短: ハック的なので汎用性が低い

- V7,画像クロップ＆ズーム,小穴補間 (V3相当),なし,6 ch,長: 物理的に小物体を大きく見せて学習を強制できる,短: 全体の空間スケール（遠近感）が狂うリスク

- V8,完全ベースライン,なし（純粋な投影のみ）,なし,6 ch,長: 前処理の純粋な効果測定（A/Bテスト）に最適,短: 実用するにはデータがスパースすぎる

- V9,特徴量エンジニアリング,小穴補間 (V3相当),なし,8 ch,長: 高さ(Z相対値)と法線(勾配)を追加し、形状認識が向上,短: モデルの入力層の変更が必要

- V10,マスクの厳密管理,小穴補間 (V3相当),なし,6 ch,長: V3を洗練させ、実測値と補間値の扱いを明確に分離した,短: 特になし（完成形の一つ）

**★BEV（鳥瞰図）**

バージョン,最大のテーマ,入力データ形式,追加されたデータ拡張 (Augmentation),特記事項・追加処理

- BEV_1  ,リアルタイム投影,.bin から都度計算,なし,実行時に毎回BEV化するため計算が遅い
  
- BEV_2  ,超高速化 (事前計算),.pt ファイル,なし,読み込みのボトルネックを解消し爆速化

- BEV_3  ,空間的なデータ拡張,.pt ファイル,水平反転、垂直反転、90度単位の回転,向きに依存しない汎用的な学習を促す

- BEV_4  ,点群特有のデータ拡張,.pt ファイル,V3 + 5%ランダム消去(Dropout) + 微小ノイズ,レーザーの欠損やセンサーノイズを再

- BEV_5  ,Cutout (矩形消去),.pt ファイル,V4 + 矩形パッチのランダム消去,車の半分が隠れた状態などを擬似的に作り出す

- BEV_6  ,学習の安定化 (正規化),.pt ファイル,V4と同等 (Cutoutは削除),各チャネルの正規化 (平均と標準偏差でスケーリング)

- BEV_7  ,マスクの不整合修正,.pt ファイル,V6と同等,ランダム消去時にマスク(mask_t)も同時に消去する修正

- BEV_8  ,高解像度化,.pt ファイル,V7と同等,読み込み先が bev_512 フォルダに（512x512解像度）

- BEV_9  ,6チャネルへの対応,.pt ファイル,V7と同等 (ノイズ追加なし),読み込み先が bev_512_6ch に。各チャネルの正規化式を調整。

- BEV_10 ,堅牢な正規化とノイズ付与,.pt ファイル,V7 + Feature Jittering (微小ノイズ),bev_512 読み込み。異常値(NaN)を防ぐための厳密なクリッピング(上限下限の設定)と、最強の正則化であるノイズ付与を復活。

**★Polar**

バージョン,最大のテーマ,入力データ形式,追加されたデータ拡張 (Augmentation),特記事項・追加処理

- Polar_1,極座標への適応と物理法則の遵守,.pt ファイル ,Azimuth Roll (ヨー回転) の追加。上下反転・90度回転の廃止。,360度ループする極座標の特性に合わせた独自のデータ拡張。現実世界であり得ない視点の反転を禁止し、学習を安定化。

**★parser**

コード名,対象の投影法,データの取り出し方式,バッチ結合処理 (Collate),特徴・設計思想と実用性

- parser1(初期のparer.py),球面投影(Range Image),.next() を呼ぶ古いイテレータ形式,PyTorchデフォルト（指定なし）,
【特徴】 最も初期のシンプルなベースライン。【実用性: ❌】 点群のようなサイズが変動するデータをPyTorchのデフォルト機能で無理やり結合しようとするため、エラーで落ちやすい不安定な設計。

- parser2,球面投影(Range Image),DataLoader を返すモダンな for ループ形式,専用関数kitti_collate_...,
【特徴】 16要素に及ぶ複雑なデータを「テンソル化するもの」と「リストのまま保持するもの」に安全に仕分ける高度な結合処理を実装。【実用性: ⭕️】 球面投影モデルへのデータ供給係として、最も堅牢で完成された設計。少数クラス（バイク等）の出現率を操作するサンプラー機能も内包可能。

- Parser_BEV1,BEV投影(鳥瞰図),.next() を呼ぶ古いイテレータ形式,PyTorchデフォルト（指定なし）,
【特徴】 parser1.py をベースに、読み込み先をBEV用データセットに変更した過渡期のコード。【実用性: ❌】 専用の結合処理がないため、BEVデータと後方互換用のダミー変数が混ざった出力をバッチ化（結合）できず、バッチサイズ2以上にした途端にクラッシュする危険性が高い。

- Parser_BEV2,BEV投影(鳥瞰図),DataLoader を返すモダンな for ループ形式,専用関数bev_collate_fn,"
【特徴】 parser2.py の堅牢な設計思想をBEV用に完全移植。Train時のみデータ拡張（フリップやノイズ等）をオンにするフラグ管理も対応。【実用性: ⭕️】 高解像度のBEVデータ（512x512, 4ch）とダミー変数を安全にバッチ化できる、現在のBEVパイプラインにおける最適解。"

- Parser_Polar1,極座標投影(Polar BEV),DataLoader を返すモダンな for ループ形式,専用関数bev_collate_fn,
【特徴】 Parser_BEV2 の堅牢な設計を継承しつつ、極座標専用データセット（SemanticKitti_Polar1）を読み込むように進化した最新コード。5チャネル（+マスクで実質6ch）の特徴量データを安全にバッチ化し、古い trainer.py が要求する「16要素のフォーマット」を崩さないようダミー変数を巧みに生成して橋渡しをする。【実用性: ⭕️】 PolarCSWinBlock などの最新の極座標モデル（SJunNet9 等）にデータを送り込むための、最も安全で完成されたパイプライン。
